# SpringMVC配置

# SpringMCV的作用

MVC框架: 解决传统基于MVC架构的JavaWeb工程的一些繁琐的操作问题，从而简化开发操作流程，规范开发架构，自动化实现一些通用功能，如，数据传递，封装，请求转发，页面标签的扩展，拦截器，异常处理。

# 添加web.xml配置文件中关于SpringMVC的配置

添加dispatcherServlet的配置

<servlet>
	<servlet-name>dispatcherServlet</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
</servlet>

<servlet-mapping>
	<servlet-name>dispatcherServlet</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>

在WebINF文件夹下创建与标签中同名的xml文件，并加入-servlet

最后的文件名为 dispatcherServlet-servlet.xml

# 为什么要命名为dispatcherServlet-servlet.xml

因为MVC默认查找的配置文件地址为: /WEB-INF/${servletName}-servlet.xml

# 配置dispatcherServlet-servlet.xml文件

<!--引入xml版本号-->
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE beans PUBLIC "-//SPRING//DTD BEAN//EN" "http://www.springframework.org/dtd/spring-beans.dtd">
<!--也可以直接在beans标签中添加xmlns-->

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans 		http://www.springframework.org/schema/beans/spring-beans.xsd">
	<!--映射配置文件-->
    <bean id="simpleUrlHandlerMapping" class="org.springframework.web.servlet.handler.SimperUrlHandlerMapping">
        <!--配置映射属性-->
        <property name="mapping">
            <props>
                <!--url访问路径以及控制器名称-->
                <prop key="/index">indexController</prop>
            </props>
        </property>
    </bean>	
    
    <!--配置控制器路径-->
    <bean id="indexController" class="com.qst.ssm.controller.IndexController"/>
</beans>

# SpringMVC视图定位

  • 在SpringMVC配置文件dispatcherServlet-servlet.xml中添加新的bean
<bean id="viewResolver" class="org.springframework.web.servlet.view.InternalResourceViewResolver">
        <property name="prefix" value="/WEB-INF/page/"/>
        <property name="suffix" value=".jsp"/>
    </bean>

此处的为视图解析器，指定视图路径

prefix 为前缀路径,指在该前缀下的所有文件为视图文件

suffix 为后缀路径，指以.jsp开始的文件为视图文件

  • 修改indexController中

ModelAndView mav = new ModelAndView("index");

将index.jsp修改为index 此处指的是将文件定位修改为视图定位，扫描所有在/WEB-INF/page/下，以.jsp为后缀的index名称的文件

  • 在WEBINF下创建page文件夹
  • 将index.jsp移入page文件夹中

public class annotationController{

    public ModelAndView handleRequest(HttpServletRequest request, HttpServletResponse response) throws Exception {

        mav.addObject("msg", "注解方式进行路径跳转");
        return mav;
    }
}

        <!--注解方式配置Controller 使用context扫描指定路径包下的类-->

# SpringMVC传递接收表单数据

首先创建bean类 Product

package com.qst.ssm.bean;

public class Product {
    private int id;
    private String name;
    private float price;

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public float getPrice() {
        return price;
    }

    public void setPrice(float price) {
        this.price = price;
    }
}

创建ProductController类

package com.qst.ssm.controller;

import com.qst.ssm.bean.Product;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.servlet.ModelAndView;

@Controller
public class ProductController {

    @RequestMapping("/addProduct")
    public ModelAndView add(Product product){
        ModelAndView mav = new ModelAndView("showProduct");
        //如果不加入下面这句话，则product会默认被当作参数加入到ModelAndView中
        mav.addObject("product",product);
        return mav;
    }
}

此处使用注解的方式声明Controller，使用@RequestMapping增加访问路径地址

public ModelAndView add(Product product){
	//这里转递过来的参数会被默认添加进ModelAndView中
	ModelAndView mav = new ModelAndView("showProduct");//这里使用视图定位的方式跳转
	//跳转到的是/WEB-INF/page/下的showProduct.jsp下
	return mav;
}

在showProduct.jsp中使用EL表达式使用SpringMVC传递的值

在addProduct.jsp中from表单的action属性中添加addProduct的访问路径

# 客户端跳转

使用SpringMVC实现客户端跳转

    @RequestMapping("/redirect")
    public ModelAndView redirectJump(){
        ModelAndView mav = new ModelAndView("index");
        mav.addObject("msg","这是从/jump跳转过来的");
        return mav;
    }

    @RequestMapping("jump")
    public ModelAndView jump(){
        ModelAndView mav = new ModelAndView("redirect:/redirect");
        return mav;
    }

访问/jump重定向到/redirect

方法为在ModelAndView构造函数中添加参数redirect:/redirect

# 解决Post提交中文乱码

在web.xml配置

	<filter> 
        <filter-name>CharacterEncodingFilter</filter-name> 
        <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class> 
        <init-param> 
            <param-name>encoding</param-name> 
            <param-value>utf-8</param-value> 
        </init-param> 
    </filter> 
    <filter-mapping> 
        <filter-name>CharacterEncodingFilter</filter-name> 
        <url-pattern>/*</url-pattern> 
    </filter-mapping>

# SpringMVC上传文件

# 上传.jpg图片

  • 首先在web.xml中配置，允许对.jpg资源的访问。

    因为在配置dispatcherServlet时，使用了/ 的url访问路径，所以所有静态资源都是默认访问不到的。需要在配置dispatcherServlet前，开放对.jpg资源的访问。

    <!--允许.jpg的图片资源的访问-->
    <servlet-mapping>
        <servlet-name>default</servlet-name>
        <url-pattern>*.jpg</url-pattern>
    </servlet-mapping>

  • 在SpringMVC的配置文件dispatcherServlet-servlet.xml中，开放对上传功能的支持

    <bean id="multipartResolver" class="org.springframework.web.multipart.commons.CommonsMultipartResolver"

  • 新建Controller UploadController

    @Controller
public class UploadController{
    @RequestMapping("/uploadImage")
    public ModelAndView upload(HttpServletRequest request, UploadedImageFile file)
            throws IllegalStateException, IOException {
        String name = RandomStringUtils.randomAlphanumeric(10);
        String newFileName = name + ".jpg";
        File newFile = new File(request.getServletContext().getRealPath("/image"), newFileName);
        newFile.getParentFile().mkdirs();
        file.getImage().transferTo(newFile);

        ModelAndView mav = new ModelAndView("showUploadedFile");
        mav.addObject("imageName", newFileName);
        return mav;
    }
}

# 支持上传.gif图片

在web.xml中增加对.gif资源的访问

<servlet-mapping>
	<servlet-name>default</servlet-name>
    <url-pattern>*.gif</url-pattern>
</servlet-mapping>

修改UploadController

@Controller
public class UploadController{
    @RequestMapping("/uploadImage")
    public ModelAndView upload(HttpServletRequest request, UploadedImageFile file)
            throws IllegalStateException, IOException {
        String name = RandomStringUtils.randomAlphanumeric(10);
        
        // 此处修改为获取file的原始文件名
        String filename = file.getImage().getOriginalFilename();
        //获取原始文件名的后缀 suffix : .jpg/.gif
        String suffix = filename.substring(filename.lastIndexOf('.'));
        //拼接文件名
        String newFileName = name + suffix;
        File newFile = new File(request.getServletContext().getRealPath("/image"), newFileName);
        newFile.getParentFile().mkdirs();
        file.getImage().transferTo(newFile);

        ModelAndView mav = new ModelAndView("showUploadedFile");
        mav.addObject("imageName", newFileName);
        return mav;
    }
}

# SpringMVC拦截器实现

  • 新建一个包com.qst.ssm.interceptor，新建一个拦截器IndexInterceptor

  • 继承HandlerInterceptorAdapter

    package com.qst.ssm.interceptor;

import org.springframework.web.servlet.ModelAndView;
import org.springframework.web.servlet.handler.HandlerInterceptorAdapter;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.util.Date;

public class IndexInterceptor extends HandlerInterceptorAdapter {

    //重写preHandle方法
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {

        System.out.println("preHandle(),在访问Controller之前被调用");

        return true;
    }

    //重写postHandle方法
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {

        System.out.println("postHandle(), 在访问Controller之后，访问视图之前被调用,这里可以注入一个时间到modelAndView中，用于后续视图显示");
        modelAndView.addObject("date","由拦截器生成的时间:" + new Date());

    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {

        System.out.println("afterCompletion(), 在访问视图之后被调用");

    }
}

  • 在dispatcherServlet-servlet.xml中配置

        <!--配置拦截器的拦截路径，以及拦截器的包路径-->
    <mvc:interceptors>
        <mvc:interceptor>
            <mvc:mapping path="/test"/>
            <bean class="com.qst.ssm.interceptor.IndexInterceptor"></bean>
        </mvc:interceptor>
    </mvc:interceptors>
