# 创建mybatis配置文件

创建mybatis-config.xml

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <typeAliases>
        <package name="com.qst.learn.bean"/>
    </typeAliases>
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="com.mysql.cj.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql:///how2j?serverTimezone=UTC&amp;allowMultiQueries=true"/>
                <property name="username" value="root"/>
                <property name="password" value="yeahqing919"/>
            </dataSource>
        </environment>
    </environments>
    <mappers>
       <!--此处添加Bean类映射文件-->
        <mapper resource="com/qst/learn/bean/Category.xml"/>
        <mapper resource="com/qst/learn/bean/Product.xml"/>
        <mapper resource="com/qst/learn/bean/Order.xml"/>
        <mapper resource="com/qst/learn/bean/OrderItem.xml"/>
    </mappers>
</configuration>

.dtd后缀文件: 文档类型定义(Document Type Definition)，可以定义xml文档结构

  1. 在中编写结果集以及CRUD操作

  2. 结果集中一对多、多对一关系的标签

    一对多,一个Category分类下对应多个产品，例如鞋分类下对应多个品牌的鞋

    <collection property="products" ofType="Product">
            <id column="pid" property="id"/>
            <result column="pname" property="name"/>
            <result column="price" property="price"/>
</collection>

    多对一,多个产品对应同一个分类，例如各种款式的包都属于包的分类

    <association property="category" javaType="Category">
            <id column="cid" property="id"/>
            <result column="cname" property="name"/>
</association>

  3. 多对多关系的理解

    多对多关系需要额外产生一张关系表，其将两个多对多关系的表联系在一起
